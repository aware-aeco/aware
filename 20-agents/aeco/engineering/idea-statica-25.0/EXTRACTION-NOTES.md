# idea-statica-25 EXTRACTION NOTES

This agent is generated from a hybrid coverage IR built by fetching the **OpenAPI YAML** specs and the **markdown SDK docs** that openapi-generator emits alongside them, both pinned to the `25.1.5.1491` tag of the `idea-statica/ideastatica-public` GitHub repository. The result is one IR per IDEA Statica major version covering **two REST APIs** (Connection API + RCS API) plus their auto-generated .NET SDK surfaces.

## Source

- **Vendor portal:** `https://developer.ideastatica.com` (DocFX-rendered markdown — only one version published at any time, no version selector for older releases)
- **Source repository:** `https://github.com/idea-statica/ideastatica-public` at tag `25.1.5.1491`
  - Connection API spec: `src/api-sdks/connection-api/clients/csharp/api/openapi.yaml`
  - Connection API SDK docs: `src/api-sdks/connection-api/clients/csharp/docs/*.md`
  - RCS API spec: `src/api-sdks/rcs-api/clients/csharp/api/openapi.yaml`
  - RCS API SDK docs: `src/api-sdks/rcs-api/clients/csharp/docs/*.md`
- **SDK version reported by README:** `25.1.5.0492` (Connection API), `25.1.5.0492` (RCS API) — both auto-generated by openapi-generator from the corresponding `openapi.yaml`
- **Source kind:** `hybrid` (OpenAPI YAML + markdown enrichment)
- **Extraction date:** 2026-05-19

## Reproduction

```bash
# From repo root — run the sidecar via stdin envelope.
echo '{"op":"coverage-extract","args":{"vendor":"idea-statica","version":"25.0","out_path":"cli-sidecar/Ingest/Output/idea-statica-25.0.ir.json"}}' \
  | cli-sidecar/bin/Release/net9.0/win-x64/publish/aware-sidecar.exe

# Regenerate the agent from the IR:
AWARE_SIDECAR=cli-sidecar/bin/Release/net9.0/win-x64/publish/aware-sidecar.exe \
  cli/target/release/aware.exe coverage generate idea-statica 25.0 \
  --from-ir cli-sidecar/Ingest/Output/idea-statica-25.0.ir.json \
  --vendor idea-statica --vertical engineering
```

The extractor lives at `cli-sidecar/Ingest/Extractors/IdeaStatica/` (Extractor.cs, OpenApiParser.cs, MemberPageParser.cs, YamlReader.cs). See `idea-statica-26.0/EXTRACTION-NOTES.md` for the full architectural rationale (this file documents version-specific stats; the architecture is identical between v25 and v26).

## Strict 50-type self-verification

A deterministic sample of 50 types was drawn from the IR (seeded from the SHA-256 of the IR file). Per-type checks: name appears in source page (markdown H1 or YAML schema key); 3 random methods have a real signature; 3 random properties have a real type (modulo the legitimate `object` exemption — see Tekla's SyncRoot pattern, mirrored here for `RcsSetting.value` and similar).

Result: **49 / 50 PASS** with 1 legitimate-`object` false positive on `RcsSetting.Value` (the YAML declares this field with no `type:` and `nullable: true`, so the IR's `type="object"` is correct per the brief's tolerance). Full output captured at `cli-sidecar/Ingest/Output/idea-statica-25-strict-verify.txt`.

## Differences from v26

Comparing v25 (87 ops + 97 schemas in Connection API, 30 + 54 in RCS API) vs v26 (91 ops + 100 schemas in Connection API, 30 + 54 in RCS API): v26 adds 4 new operations + 3 new schemas to the Connection API. RCS API is unchanged between v25 and v26.

The new v26 Connection API operations (sampled from the tag-diff): `CreateEmptyConnection`, `PublishConnection`, `Propose` (template proposals), `AddCrossSection`. Otherwise the surfaces overlap substantially — the SDK is backward-compatible at the v25→v26 boundary.

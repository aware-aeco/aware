# Bucketing NC files for the shop floor

The detailer's Friday-afternoon problem: 800 pieces in a phase, four
production machines, and the saw line operator wants only saw files,
the drill operator only drill files. Sending them all every file is a
recipe for the wrong file being loaded.

## Solution: `group-by-machine`

```yaml
- id: nc-export
  agent: tekla
  command: nc-export-phase       # v0.16 — promoted curated verb
  inputs:
    phase: 2
    output-dir: '{{ run.tmp-dir }}/raw-nc'

- id: convert
  agent: peddinghaus-translator
  command: to-peddimat
  inputs:
    input-dir: '{{ run.tmp-dir }}/raw-nc'
    output-dir: '{{ run.tmp-dir }}/peddimat'
    machine-type: beam-line     # full controller-friendly output

- id: bucket
  agent: peddinghaus-translator
  command: group-by-machine
  inputs:
    input-dir: '{{ run.tmp-dir }}/peddimat'
    output-dir: '\\shop-floor\queue\phase-2'
    machines: [saw, drill, plasma]
  safety:
    transaction-group: nc-issue
    snapshot: false  # shop-floor queue is replace-not-merge
```

Output structure:

```
\\shop-floor\queue\phase-2\
  saw\
    A-100-1.asc
    A-100-2.asc
    ...
  drill\
    A-100-1.asc
    A-100-2.asc
    ...
  plasma\
    A-101-3.asc
    ...
  manifest.csv
```

## What's in the manifest

```csv
piece_mark,profile,length_mm,saw,drill,plasma,coper
A-100-1,UB 305x165x40,6200,saw\A-100-1.asc,drill\A-100-1.asc,,
A-100-2,UB 305x165x40,6200,saw\A-100-2.asc,drill\A-100-2.asc,,
A-101-3,PL 12x300,1800,,,plasma\A-101-3.asc,
```

Empty cells = piece doesn't need that machine. The shop-floor traveller
sheet is generated by joining this CSV against the order register.

## Bucketing logic

A piece goes to the `saw` queue if:
- It has an `FB` (saw-stop) block — comes from `to-peddimat --machine-type saw|beam-line`
- OR its `length-mm` is non-zero (i.e. it needs cutting at all)

A piece goes to the `drill` queue if it has any `BO` blocks (holes).

A piece goes to the `plasma` queue if it has `KO` (notch) or non-rectangular `AK` (outer contour).

A piece goes to the `coper` queue if its profile is an I-shape AND it
has a `KO` at a beam end (typical web cope).

Tiny pieces (< 200mm) with one hole skip everything and ship as-is to
welding — the bucketer detects and puts them in a `direct-weld` queue
if requested.

## Once-a-week vs continuous

A fab shop running this once-a-week (Friday afternoon) needs the
queues clobbered (`replace` mode). A shop running it continuously
(per Tekla save) needs the queues *appended* — that's a future
follow-up (`mode: append` flag on `group-by-machine`). For v0.16
the verb replaces; manage incremental queues by detailer-side staging.

## What if the shop runs SDS/2 / Advance Steel instead?

Those tools emit DSTV too (sometimes in a slightly different dialect).
This agent reads any DSTV that conforms to DIN 9404. If you hit a
dialect mismatch, send the offending file to the project's BIM
authority — every dialect we add to the parser is a slight maintenance
tax.

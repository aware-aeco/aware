# v0.30 — Host-Agent 100%-Coverage Rework Design

> **Status:** Brainstormed + approved 2026-05-18 across all 7 sections. Implementation plan follows via `superpowers:writing-plans`.
>
> **Goal:** Re-engineer AWARE's raw host-agent coverage from "NuGet reflection only (API surface, zero behavioral knowledge)" to "100% vendor-documented API surface + behavioral notes". Concrete failure that motivated this rework: building the v0.29 Tekla sidecar, I burned hours trying to find `Operation.ExitTekla()` via reflection — only to find by reading developer.tekla.com that **no external exit API exists at all**. The substrate had reflected the methods but not the documentation telling you when (and whether) you can use them. The same gap applies to every other raw host agent.

---

## 1 — Architecture

```
            VENDOR DOCS (per host)
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
  per-vendor extractor (cli-sidecar/Ingest/<vendor>/)
  ├── tekla        scrapes developer.tekla.com
  ├── revit        parses RevitSDK CHM + RevitAPI.xml
  ├── sketchup     scrapes ruby.sketchup.com
  ├── autocad      parses ObjectARX SDK help files
  ├── archicad     scrapes Graphisoft DevPortal
  ├── rhino        scrapes McNeel developer.rhino3d.com
  ├── grasshopper  scrapes McNeel developer.rhino3d.com/api
  ├── dynamo       scrapes Autodesk Dynamo Forge API docs
  ├── allplan      scrapes Allplan PythonParts API docs
  ├── navisworks   parses Navisworks SDK CHM + docs
  ├── idea-statica scrapes ideastatica.com developer center
  ├── tsd          scrapes Tekla Tedds developer pages
  ├── solibri      scrapes Solibri OSO REST API
  └── bluebeam     scrapes Bluebeam Studio API
                    │
                    ▼
        ┌──── Uniform Intermediate Representation (IR) ────┐
        │  { type, namespace, kind, summary, remarks,      │
        │    methods, properties, events, enums }          │
        └──────────────────────────────────────────────────┘
                    │
                    ▼
        host-agnostic generator (single tool)
                    │
        ┌───────────┼────────────┐
        ▼           ▼            ▼
   manifest.yaml  skills/*.md  catalog/*.json
   (compact)      (narrative)   (structured)
                    │
                    ▼
        codex-reviewer pass per agent
        (full type enumeration + sampled deep-check)
                    │
                    ▼
        commit per agent (one PR per host)
```

**Key idea:** decouple extractors from generators via a uniform IR. Per-vendor extractors absorb format peculiarities (scraping, CHM parsing, PDF text extract). Generator and codex reviewer are vendor-agnostic — write once, reuse 14× across hosts.

## 4 — Execution plan (3 phases)

### Phase A — Foundation (sequential, ~3 days)

Pre-requisite for everything else.

| Task | Deliverable |
|---|---|
| A1 | `cli-sidecar/Ingest/Schema/host-coverage.schema.json` |
| A2 | `cli-sidecar/Ingest/Generator/` — IR → manifest + skills + catalog |
| A3 | `aware build agent --from-coverage <ir.json>` (Rust CLI wiring) |
| A4 | `docs/superpowers/specs/host-coverage-review-protocol.md` |

### Phase B — Extractors (massively parallel, ~5 days wall-clock)

One subagent per vendor family, all dispatched simultaneously after Phase A.

Each subagent's deliverables:
1. Extractor code at `cli-sidecar/Ingest/<vendor>/`
2. IR file(s) at `cli-sidecar/Ingest/Output/<vendor>-<version>.ir.json`
3. Generated agent at `20-agents/aeco/<vertical>/<host>-<version>/`
4. Self-verification report: random 50-page sample, % present in output
5. Diff vs old reflection-only agent (concrete proof of new coverage)

### Phase C — Codex review (parallel, ~1-2 days)

Per agent, a `codex-rescue` subagent runs the verification protocol (Section 5). Failed review → re-runs the loop. No human approvals on shortcuts.

### Phase D — Ship (sequential, ~1 day)

Per host: one PR with extractor + IR + regenerated agent + codex review report. Merge order:
1. TSD (smallest, dial in review pattern)
2. Bluebeam, Solibri
3. SketchUp
4. Tekla 2025, Tekla 2026
5. Rhino 7/8 + Grasshopper 7/8
6. ArchiCAD 28/29
7. IDEA Statica, Allplan, Dynamo
8. Navisworks
9. AutoCAD 2025/2026
10. Revit 2025/2026 (last — biggest review surface)

**Total wall-clock: ~10 days.** Versus ~30 sequential.

## 5 — Codex reviewer protocol

The verification engine. **Pass = full type enumeration coverage + deep-check sample passes.**

### Per-agent review steps

**Step 1 — Type enumeration (FULL, not sampled)**
- Fetch vendor type-index URL
- Extract every type name + namespace
- Compare against `catalog/*.json`
- **Fail criterion:** any vendor-indexed type missing → list emitted, blocks merge
- Result: `missing_types_count` (must be 0)

**Step 2 — Deep-check sampling (N=50 random types)**
- Pseudorandom seed = IR sha (deterministic re-runs)
- Fetch each sampled type's doc page
- Parse documented methods/properties/events/constructors/enum values
- Compare each member name + signature against catalog entry
- **Fail criterion:** any documented member missing OR signature mismatch
- Result: `deep_check_pass_rate` (must be 100/50)

**Step 3 — Behavioral spot-check (N=20 methods)**
- Random subset of deep-checked methods
- Compare catalog's `remarks` / `summary` text presence against vendor page prose
- **Fail criterion:** any sampled method with empty `remarks` when vendor has prose
- Result: `behavior_present_rate` (must be 100/20)

**Step 4 — Schema validation**
- Validate every artefact against `host-coverage.schema.json`
- **Fail criterion:** any required field empty/missing or wrong type
- Result: `schema_violations` (must be 0)

### Review output (committed alongside agent)

`20-agents/aeco/<vertical>/<host>-<version>/COVERAGE-REVIEW.md` — contains verdict, sample lists, IR sha for reproducibility, re-run command.

### Re-run loop

Failed review → subagent reads gap report → fixes extractor → regenerates IR + agent → codex re-reviews. Loop until PASS.

## 6 — Versioning + repo layout

### Rule 1 — Curated and raw agents stay separate

| Agent kind | Shape | Source | Example |
|---|---|---|---|
| Curated (hand) | Stable verbs across versions, judgment skills | `sdk-target: 2025+` | `tekla` (22 verbs, 33 skills) |
| Raw (generated) | Per-version pinned, 100% vendor surface | One IR per host-version | `tekla-2025`, `tekla-2026` |

v0.30 regenerates the **raws**. Curated agents stay untouched. Apps compose with whichever fits.

### Rule 2 — One raw agent per host-version

| Host | v0.30 versions |
|---|---|
| Tekla | 2025.0, 2026.0 |
| Revit | 2025, 2026 |
| AutoCAD | 2025, 2026 |
| SketchUp | 2025, 2026 |
| Rhino | 7, 8 |
| Grasshopper | 7, 8 |
| ArchiCAD | 28, 29 |
| IDEA Statica | 25, 26 |
| TSD | 25, 26 |
| Allplan | 2024, 2025 |
| Dynamo | 4.1.0, 4.1.1 |
| Navisworks | 2026 |
| Solibri, Bluebeam | single each (REST APIs) |

Existing raw agents replaced **in-place** (same name, same path). `version` becomes the doc-ingest version (`0.30.0`); `sdk-target` keeps the literal vendor version.

### Rule 3 — IR files committed to git

`cli-sidecar/Ingest/Output/<host>-<version>.ir.json` — ~3-5 MB per host-version, ~50 MB total.

Why commit:
- **Reproducibility:** anyone can regen agent from IR with one command, no re-scrape needed
- **Auditability:** codex review references IR sha
- **Diff visibility:** re-extraction next year shows exact vendor-API delta

### Resulting agent folder shape

```
20-agents/aeco/engineering/tekla-2026/
├── manifest.yaml                       # compact verb catalog
├── skills/                             # narrative per namespace
│   ├── tekla-structures-model.md
│   ├── tekla-structures-model-operations.md
│   └── ... (~50 namespace skills)
├── commands/                           # stable-verb pages (mostly from curated layer)
├── catalog/                            # structured per-type
│   ├── tekla.structures.model.Model.json
│   ├── tekla.structures.model.Operations.Operation.json
│   └── ... (~850 type files)
└── COVERAGE-REVIEW.md                  # codex verdict + spot-check
```

## 7 — Success criteria

v0.30 ships when:

1. **Every host listed in §3 has a regenerated raw agent** with COVERAGE-REVIEW.md showing PASS verdict
2. **Every COVERAGE-REVIEW.md has:** missing_types_count=0, deep_check_pass_rate=100%, behavior_present_rate=100%, schema_violations=0
3. **IR files are committed** to `cli-sidecar/Ingest/Output/`
4. **The Tekla close-app gap is closed by data** — the regenerated tekla-2026 catalog has explicit "no external exit API" guidance derived from the actual docs page, so the next person hitting this won't burn cycles guessing
5. **`aware report substrate` shows expanded counts** (was 24,747 commands across reflected agents; will be similar count BUT each with vendor description + behavioral notes; skills count grows from 505 to ~5,000+)

## 9 — Risks

| Risk | Mitigation |
|---|---|
| Vendor docs site re-layouts (Trimble redesigns developer.tekla.com mid-extract) | Per-vendor extractor has CSS selector tests; alert on failure |
| CHM extraction on macOS/Linux | Revit/AutoCAD extractors are Windows-only (matches the platform they're for); fine. |
| Login-walled docs (SketchUp C SDK was already skipped; APS portal partly behind login) | Use the public surface; document gaps in COVERAGE-REVIEW.md as known-uncovered |
| Codex review false negatives ("missing" type that vendor moved/renamed) | Per-host extractor maintains a `aliases.yaml` listing renames; review honours it |
| Repo size from committed IR | ~50 MB total is acceptable; if it grows past 200 MB, switch to git LFS |

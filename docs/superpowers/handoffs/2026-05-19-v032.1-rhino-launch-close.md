# v0.32.1 — rhino launch + close (2026-05-19)

Stacked on top of [v0.32 PR #66](https://github.com/aware-aeco/aware/pull/66). Brings aware-rhino to 5-verb parity with cli-tekla.

## What shipped

- **`cli-rhino/Verbs/Launch.cs`** — spawns Rhino 8 with `/nosplash` and optional `/runscript=_StartScriptServer`. Auto-discovers `Rhino.exe` (standard install or HKLM registry); accepts `rhino_exe` override. Optional `model_path` to open a 3dm. Returns `{status, host, host_pid, verb_result:{model_path, script_server, rhino_exe, note}}`. Does NOT wait for Script Server readiness — caller polls `list-instances`.
- **`cli-rhino/Verbs/Close.cs`** — two modes:
  - **Clean** (default): issues `_-Exit _N` (or `_-Exit _Y` if `save: true`) through `rhinocode command`. Polls PID for exit; returns `exited_cleanly` flag. Exit 0 = process exited; exit 1 = process still alive after 30s.
  - **Force** (`force: true`): resolves target PID, `Process.Kill(entireProcessTree: true)`. Explicit data-loss acknowledgement.
- **Program.cs** — wires the 2 new verbs into dispatch. Launch doesn't need rhinocode (it spawns Rhino directly), so client discovery is now lazy.
- **`rhino-8/manifest.yaml`** — adds `launch` and `close` command entries.
- **7 new unit tests** (4 Launch + 3 Close) using a tiny .cmd stub binary for spawn validation. Tests now total **29 green** (was 22). Runs without Rhino installed.

## Run the launch+close lifecycle (live, your task)

```powershell
$awareRhino = "cli-rhino\bin\Release\net10.0-windows\win-x64\publish\aware-rhino.exe"

# 1. Launch a fresh Rhino with a model + auto Script Server
'{"verb":"launch","model_path":"C:\\Models\\hello.3dm"}' | & $awareRhino --json-stdin
# Output includes "host_pid". Wait ~10s for Script Server to come up.

# 2. Confirm via list-instances
'{"verb":"list-instances"}' | & $awareRhino --json-stdin

# 3. Run a quick exec
'{"verb":"exec","code":"return Rhino.RhinoDoc.ActiveDoc.Objects.Count;"}' | & $awareRhino --json-stdin

# 4. Send a status message
'{"verb":"send-status","message":"hello from AWARE"}' | & $awareRhino --json-stdin

# 5. Clean shutdown (no save)
'{"verb":"close"}' | & $awareRhino --json-stdin
```

For force-kill: `'{"verb":"close","force":true,"pid":<PID>}'`. For save-and-exit: `'{"verb":"close","save":true}'`.

## What's NOT in this PR

- Live drill of launch+close (no Rhino on the build machine; same constraint as v0.32)
- Cross-instance routing edge cases (e.g., user has multiple Rhino versions installed — single Rhino 8 install assumed)
- Mac / Linux (rhinocode + Rhino.exe paths are Windows-specific)
